# StrangeSortTest 测试规范

## 目标

1. `StrangeSortTest` 的首要职责是防止公开行为漂移和功能实现错误。
2. 测试要以 `specs/StrangeSort 开发规范.md` 内算法行为协议为准，不要只按当前实现去写断言。

## 通用要求

1. 任何算法改动，至少覆盖对应算法的数组重载和 `List<T>` 重载。
2. 任何协议变更，必须在同一次改动中同步更新规范和测试。
3. 禁止为了让测试通过而在实现中硬编码输入、长度、随机种子或比较结果。
4. 随机相关测试必须显式控制随机种子，不要依赖 `Random.Shared` 的即时输出。
5. 异常测试除了断言异常类型，还要断言输入是否被修改。

## 边缘条件测试指南

1. 优先覆盖空输入、单元素输入、重复元素、已满足目标条件的输入和最小可复现输入。
2. 覆盖 `null` 参数、`comparer == null`、`random == null`、非法枚举值、不可比较类型和溢出路径。
3. 区分数组重载与 `List<T>` 重载的输入修改语义、返回引用语义和异常后状态。
4. 如果规范声明了相对顺序、元素多重集、子序列性质、固定过滤区间或固定结果长度，这些都应有直接断言。
5. 文档中的调用示例应尽量有对应测试，避免示例和真实行为脱节。

## 各算法重点

### StalinSort

1. 断言输出是输入子序列，且相对顺序保持不变。
2. 断言数组重载在长度 `0` 或 `1` 时返回原引用，在更长输入时返回新数组。
3. 覆盖默认比较器不可用时的异常，以及 `List<T>` 异常后可能部分修改。

### ThanosSort

1. 断言结果始终是有序子序列。
2. 覆盖 `FloorHalf` 和 `CeilingHalf` 两种删除策略。
3. 使用固定种子验证数组重载可复现、列表重载可复现，以及两者结果一致。
4. 覆盖非法 `RemovalCountStrategy` 和显式 `random == null`。

### EpsteinSort

1. 断言结果只包含 `[0, 18)` 内的值。
2. 对浮点类型覆盖 `NaN`、`-Infinity` 和 `+Infinity`。
3. 断言过滤不受比较器影响，比较器只影响排序结果。
4. 覆盖排序异常时 `List<T>` 可能已经完成过滤的情况。

### BrezhnevSort

1. 断言结果只由输入长度决定，与输入值无关。
2. 断言数组重载始终返回新数组。
3. 覆盖目标类型可表示范围不足时的 `OverflowException`。
4. 覆盖 `List<T>` 溢出时可能部分改写的情况。

### TrumpSort

1. 断言结果长度不变，元素多重集不变。
2. 使用固定种子验证数组重载可复现、列表重载可复现，以及两者结果一致。
3. 断言数组重载始终返回新数组。
4. 覆盖显式 `random == null` 和随机源异常传播。
