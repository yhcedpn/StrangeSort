# StrangeSort 开发指引

## 开发环境

用户基于 Visual Studio 的解决方案开发，解决方案内有下列项目：

* StrangeSort 项目。类型为 C# 类库。启用了 AOT 兼容性检查。包含所有核心代码逻辑。
* StrangeSortTest 项目。类型为 C# xUnit 测试项目。启用了 AOT 发布。用于对 StrangeSort 项目代码逻辑进行测试。

## 规范文件

规范文件位于 specs/ 目录中，其内的每个文件内包含必须遵守的开发规范。如果你发现各个规范文件之间有冲突，必须告诉用户。

## 计划文件

位于 plans/ 目录中，包含所有的开发计划文件。每个文件内包含一个或多个功能实现步骤，每个步骤必须有清晰的目标。计划文件必须足够完善，可以直接指导功能实现。

## 开发计划注意事项

当用户要求你制定一个开发计划，用户可能会提供非常模糊的需求，你必须就此计划的方方面面对用户进行深入细致的询问（可用 question 工具询问用户），直到达成共识。逐一梳理设计树的每个分支，逐一解决决策之间的依赖关系。如果某个问题可以通过查看代码库来解答，那就去调用 Explore 子代理查看代码库。

针对每个问题，提供建议答案。

如果有必要，可以用 websearch 工具联网搜素信息，或通过 webfetch 工具直接获取某个网页的内容。

## 功能实现注意事项

当用户要求你把某个计划文件落地实现，只阅读用户指定的计划文件而不阅读别的计划文件。
在落地实现计划文件前时，必须读取所有规范文件并遵守其中的规则。
你可以根据开发计划文件中的步骤，调用 todowrite 工具管理待办事项，将具体的实现工作交给 General 子代理，分步完成任务。
