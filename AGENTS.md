# StrangeSort 开发指引

## 开发环境

用户基于 Visual Studio 的解决方案开发，解决方案内有下列项目：

* StrangeSort 项目。类型为 C# 类库。启用了 AOT 兼容性检查。包含所有核心代码逻辑。
* StrangeSortTest 项目。类型为 C# xUnit 测试项目。启用了 AOT 发布。用于对 StrangeSort 项目代码逻辑进行测试。

## 技能文件

技能文件位于 skills/ 目录中。若提到阅读技能文件，在这个目录中查找。

### 技能：改进代码库架构

当用户提到“改进代码库架构”，阅读 技能-改进代码库架构.md 这个文件，按照里面的要求执行。

### 技能：烤我

当用户提到“烤我”，阅读 技能-烤我.md 这个文件，按照里面的要求执行。

### 技能：与 Git 交互

当用户要求你与 Git 交互，例如执行 Git 提交（也包括制定提交计划），附加 Git 标签（也包括制定附加标签的计划），变更 Git 提交历史，推送本地变更到远端仓库，阅读 技能-与 Git 交互.md 这个文件，按照里面的要求执行。

### 技能：与 GitHub 交互

当用户要求你与 GitHub 交互，例如访问 GitHub 仓库，将本地变更推送到 GitHub 仓库，阅读 技能-与 GitHub 交互.md 这个文件，按照里面的要求执行。

## 规范文件

规范文件位于 specs/ 目录中，其内的每个文件内包含必须遵守的开发规范。如果你发现各个规范文件之间有冲突，必须告诉用户。

## 计划文件

位于 plans/ 目录中，包含所有的开发计划文件。

每个文件内包含一个或多个功能实现步骤，每个步骤必须有清晰的目标。每个步骤尽可能包含较少的变更，不要在一个步骤内进行大型变更。
计划文件必须足够完善，可以直接指导功能实现。

计划文件内不应该引用其他计划文件。
当用户要求你把制定好的开发计划写入到文件时，除非用户明确要求你写入到已有的计划文件，否则你必须写入到新计划文件。
除非用户要求，否则忽略计划文件目录内的计划文件（除了用户提到的文件），将它们视为历史计划。制定新计划时，不必变更旧的计划。

## 功能实现注意事项

当用户要求你把某个计划文件落地实现：

只阅读用户指定的计划文件而不阅读别的计划文件。
在落地实现计划文件前，必须读取所有规范文件并遵守其中的规则。
在落地实现计划文件时，你可以根据开发计划文件中的步骤，调用 todowrite 工具管理待办事项，将每一步具体的实现工作交给 General 子代理，分步完成任务。
