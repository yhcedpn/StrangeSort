# TrumpSorter ShuffleAndWin 实现计划

## 目标

为项目新增一个带随机性的“政治梗排序”算法入口。该算法的实际语义不是“把元素排成某种有序结果”，而是对输入序列执行一次随机重排，因此更准确地说，它是一个单次 shuffle 帮助方法。

本次实现需要与当前代码库风格保持一致：

1. 公开类型采用独立命名空间和独立静态类型，不修改现有 `StalinSort`、`ThanosSort`、`EpsteinSort`、`BrezhnevSort` 的公开契约。
2. 数组重载不修改输入数组，而是返回新的结果数组。
3. 列表重载原地修改输入列表，保持与现有项目对 `List<T>` 的处理风格一致。
4. 随机性通过 `Random` 提供；默认重载使用 `Random.Shared`，并额外提供可显式传入 `Random` 的重载以支持复现和测试。
5. 行为必须可测试，且测试重点应放在可复现性和公开不变式上，而不是把某个默认随机轨迹当作长期契约。

## 设计定稿

### 类型与命名

1. 在 `StrangeSort/TrumpSort/` 目录下新增 `TrumpSorter.cs`。
2. 命名空间使用 `StrangeSort.TrumpSort`。
3. 公开类型名为 `TrumpSorter`。
4. 公开方法名为 `ShuffleAndWin`。
5. 类型和方法注释中必须明确说明：该能力会随机重排输入内容，并不保证结果有序；“Win”只是命名梗，不表示算法成功找到某种最优顺序。

### 公开 API

计划提供以下公开重载：

```csharp
public static T[] ShuffleAndWin<T>(T[] values)
public static T[] ShuffleAndWin<T>(T[] values, Random random)

public static void ShuffleAndWin<T>(List<T> values)
public static void ShuffleAndWin<T>(List<T> values, Random random)
```

本次不提供以下公开能力：

1. 不提供 `IComparer<T>` 或 `Comparison<T>` 参数，因为此算法不比较元素，不依赖“有序”定义。
2. 不提供“持续打乱直到有序”的版本，因为那会把语义从单次随机重排改成另一种可能长时间运行的随机算法。
3. 不提供最大尝试次数、轮数统计、胜率统计等扩展参数，因为单次 shuffle 不存在终止性问题。
4. 不提供 `IList<T>`、`Span<T>`、`Memory<T>` 或扩展方法重载，首版只覆盖项目现有主流入口 `T[]` 与 `List<T>`。
5. 不返回打乱前后索引映射、交换次数或包装结果对象。

### 核心语义

1. 算法对输入序列执行一次 Fisher-Yates shuffle。
2. 每个位置在等概率前提下参与最终排列，结果应是输入元素的一个随机排列。
3. 算法不会删除元素、不会新增元素、不会修改元素值本身。
4. 对包含重复值的输入，结果允许出现与输入视觉上相同的排列；测试时只能验证元素多重集一致，不能要求“看起来一定变化”。
5. 空输入和单元素输入应直接成功，且不要求随机源实际参与抽样。
6. 由于语义是“单次随机重排”，因此不存在“是否会一直跑不完”的问题。

### 异常与输入契约

1. 数组或列表输入为 `null` 时，抛出 `ArgumentNullException`。
2. 带 `Random` 参数的重载传入 `null` 时，抛出 `ArgumentNullException`。
3. 默认重载使用 `Random.Shared`。
4. 数组重载在随机过程中若抛异常，不得修改输入数组。
5. 列表重载允许在原地处理过程中因随机源异常而出现部分修改，不要求回滚；需要在文档注释中明确说明这一点。
6. 对外不捕获并包装随机源抛出的原始异常。

### 推荐实现方式

1. 数组版本先复制输入数组，再对副本执行原地 Fisher-Yates shuffle，最后返回副本。
2. 列表版本直接在输入列表上执行原地 Fisher-Yates shuffle。
3. 内部共享一个最小私有辅助方法，用于按给定 `Random` 对索引区间执行交换。
4. 除了必要的空值校验和长度短路外，不引入额外策略类型或随机抽象，保持实现最小化。

## 实现步骤

### 步骤 1：建立公开类型与行为说明

目标：建立 `TrumpSorter` 的目录、文件、命名空间、XML 注释和公开重载骨架，使其 API 语义在代码层面一眼可见。

实现要求：

1. 新增 `StrangeSort/TrumpSort/TrumpSorter.cs`。
2. 定义数组和列表的默认随机源重载与显式 `Random` 重载。
3. 在类型和方法注释中明确写出：
4. 这不是传统意义上的“排序”。
5. 该算法只执行一次随机重排。
6. 数组重载不会修改输入数组。
7. 列表重载会原地修改输入列表。
8. 列表重载在随机源抛异常时可能已经部分修改且不会回滚。

完成标准：

1. 代码文件可编译。
2. 公开 API 名称与计划一致。
3. 注释完整且使用简体中文。

### 步骤 2：实现数组重载的随机重排逻辑

目标：完成数组版本 `ShuffleAndWin` 的实际行为，确保返回值是输入元素的随机排列，且输入数组保持不变。

实现要求：

1. 对 `values` 执行空值校验。
2. 对带 `Random` 参数的重载执行空值校验。
3. 对于长度为 `0` 或 `1` 的数组直接返回原数组或等价结果，不引入无意义交换。
4. 复制输入数组后，对副本执行一次 Fisher-Yates shuffle。
5. 不在原数组上做任何原地写入。
6. 返回长度与输入一致的结果数组。

完成标准：

1. 数组输入不会被修改。
2. 返回结果与输入具有相同长度。
3. 返回结果与输入包含相同的元素多重集。

### 步骤 3：实现列表重载的原地随机重排逻辑

目标：完成列表版本 `ShuffleAndWin` 的原地处理，尽量复用与数组版本一致的交换语义，同时保持项目现有 `List<T>` API 风格。

实现要求：

1. 对 `values` 执行空值校验。
2. 对带 `Random` 参数的重载执行空值校验。
3. 对于长度为 `0` 或 `1` 的列表直接返回。
4. 在原列表上执行一次 Fisher-Yates shuffle。
5. 不替换列表实例，不创建额外公开结果对象。
6. 随机源抛异常时仅传播异常，不保证列表回滚。

完成标准：

1. 列表会被原地修改。
2. 调用后列表长度保持不变。
3. 调用后列表与调用前包含相同的元素多重集。

### 步骤 4：补齐测试用例并锁定公共语义

目标：在 `StrangeSortTest/TrumpSort/` 下新增测试文件，覆盖输入校验、随机复现性、容器差异和异常行为，确保后续重构不会破坏公开约定。

建议测试清单：

1. `null` 数组输入抛出 `ArgumentNullException`。
2. `null` 列表输入抛出 `ArgumentNullException`。
3. 带 `Random` 参数的数组重载传入 `null` 抛出 `ArgumentNullException`。
4. 带 `Random` 参数的列表重载传入 `null` 抛出 `ArgumentNullException`。
5. 空数组返回空结果。
6. 空列表处理后保持为空。
7. 单元素数组返回等价内容，且输入数组不变。
8. 单元素列表处理后内容不变。
9. 数组版本在固定 seed 下两次调用结果一致。
10. 列表版本在固定 seed 下两次调用结果一致。
11. 数组和列表在相同输入、相同 seed 下产生相同内容。
12. 数组版本返回结果长度等于输入长度，且输入数组内容不变。
13. 列表版本处理后长度保持不变，且仍是同一个列表实例。
14. 含重复值输入时，结果与输入的元素多重集一致。
15. 默认随机源重载只验证公开不变式，不断言具体排列结果。
16. 数组重载在随机源抛异常时不修改输入数组。
17. 列表重载在随机源抛异常时传播异常。

完成标准：

1. 测试命名与现有测试风格一致。
2. 测试覆盖数组与列表两个公开入口。
3. 测试通过注入 `Random` 控制复现，不依赖脆弱的默认随机轨迹。

### 步骤 5：执行验证并检查对外一致性

目标：通过构建和测试确认新增功能可用，并检查命名、注释和行为描述是否与既有代码风格一致。

验证要求：

1. 执行 `dotnet build "StrangeSort.slnx"`。
2. 执行 `dotnet test "StrangeSort.slnx"`。
3. 确认新增命名、注释和异常约定与现有 `ThanosSorter`、`StalinSorter` 的表述风格保持接近。
4. 确认没有无意中引入比较器参数、最大尝试次数或其他超出本计划的扩展 API。

完成标准：

1. 构建通过。
2. 全量测试通过。
3. `TrumpSorter` 的数组与列表行为符合本计划约定。

## 验收口径

功能完成后，应满足以下验收条件：

1. 仓库中存在新的 `StrangeSort.TrumpSort` 命名空间与 `TrumpSorter` 类型。
2. 用户可以对 `T[]` 和 `List<T>` 调用 `ShuffleAndWin`。
3. 数组调用后输入保持不变，返回值为随机排列结果。
4. 列表调用后原列表被原地随机重排。
5. 默认重载使用 `Random.Shared`，显式 `Random` 重载可用于复现与测试。
6. 结果不会删除或新增元素，只会改变元素排列顺序。
7. 行为由测试稳定覆盖，尤其是空输入、单元素、固定 seed 复现、数组不变性、列表原地修改和异常传播这些关键语义。
