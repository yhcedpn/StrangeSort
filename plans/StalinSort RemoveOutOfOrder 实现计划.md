# StalinSort RemoveOutOfOrder 实现计划

## 目标

为项目新增一个确定性的“删除式排序”算法。该算法面对数组或列表时，从左到右遍历元素；如果当前元素不满足相对于最近一个保留元素的顺序要求，则直接删除当前元素。最终结果不是重排后的完整排序结果，而是一个保持原有相对顺序的有序子序列。

本次实现需要与当前代码库风格保持一致：

1. 公开类型采用独立命名空间和独立静态类型，不修改现有 `ThanosSort` 的公开契约。
2. 数组重载不修改输入数组，而是返回新的结果数组。
3. 列表重载原地修改输入列表，保持与 `List<T>.Sort` 以及现有 `ThanosSorter` 列表重载相近的使用感受。
4. 公开比较器参数仅使用 `IComparer<T>?`，当其为 `null` 时回退到 `Comparer<T>.Default`。
5. 行为必须可测试，且测试用例能够稳定验证核心语义。

## 设计定稿

### 类型与命名

1. 在 `StrangeSort/StalinSort/` 目录下新增 `StalinSorter.cs`。
2. 命名空间使用 `StrangeSort.StalinSort`。
3. 公开类型名为 `StalinSorter`。
4. 公开方法名为 `RemoveOutOfOrder`，避免把该能力伪装成传统的“重排排序”。

### 公开 API

计划提供以下公开重载：

```csharp
public static T[] RemoveOutOfOrder<T>(T[] values)
public static T[] RemoveOutOfOrder<T>(T[] values, IComparer<T>? comparer)

public static void RemoveOutOfOrder<T>(List<T> values)
public static void RemoveOutOfOrder<T>(List<T> values, IComparer<T>? comparer)
```

本次不提供以下公开能力：

1. 不提供 `Comparison<T>` 重载。
2. 不提供 `IList<T>` 公共重载。
3. 不返回删除数量、删除索引或其他包装结果。
4. 不引入策略枚举，因为本次算法的删除规则已经固定为“删除当前违序元素”。

### 顺序语义

1. “有序”定义为：对每个保留元素序列中的相邻元素 `previous` 和 `current`，都满足 `comparer.Compare(previous, current) <= 0`。
2. 允许相等元素同时保留。
3. 算法按从左到右顺序扫描。
4. 若当前元素相对于最近一个保留元素违序，则删除当前元素。
5. 保留元素必须保持输入中的原有相对顺序。

示例：

1. `[3, 1, 2]` 处理结果为 `[3]`。
2. `[1, 3, 2, 4]` 处理结果为 `[1, 3, 4]`。
3. 对逆序比较器而言，结果应改为满足该比较器定义的有序子序列。

### 异常与输入契约

1. 数组或列表输入为 `null` 时，抛出 `ArgumentNullException`。
2. 当 `comparer` 为 `null` 时，使用 `Comparer<T>.Default`。
3. 如果元素类型默认比较器无法比较，只有在实际需要比较时才会抛出原始异常。
4. 空输入和单元素输入无需进行比较，应直接成功。
5. 数组重载在比较过程中抛异常时，不得修改输入数组。
6. 列表重载允许原地处理过程中因比较器异常而出现部分修改，不要求回滚；需要在文档注释中明确说明这一点。

## 实现步骤

### 步骤 1：新增核心公开类型与最小实现框架

目标：建立 `StalinSorter` 的文件、命名空间、XML 注释和公开重载骨架，使其目录结构、命名和说明文档与现有项目风格一致。

实现要求：

1. 新增 `StrangeSort/StalinSort/StalinSorter.cs`。
2. 为数组和列表分别提供默认比较器重载与 `IComparer<T>?` 重载。
3. 在类型和方法注释中明确说明：
4. 这是通过删除违序元素得到有序子序列的算法。
5. 数组重载不会修改输入数组。
6. 列表重载会原地修改输入列表。
7. 列表重载在处理中若比较器抛异常，可能已经部分修改且不会回滚。

完成标准：

1. 代码文件可编译。
2. 公开 API 名称与计划一致。
3. 注释完整且使用简体中文。

### 步骤 2：实现数组重载的确定性剪枝逻辑

目标：完成数组版本 `RemoveOutOfOrder` 的实际行为，确保结果为稳定的有序子序列，且输入数组保持不变。

实现要求：

1. 对 `values` 执行空值校验。
2. 将 `comparer ??= Comparer<T>.Default` 作为统一比较器入口。
3. 对于长度为 `0` 或 `1` 的数组直接返回原数组或等价结果，不引入无意义比较。
4. 使用单次线性扫描决定保留元素。
5. 不在原数组上做原地删改。
6. 生成仅包含保留元素的新数组并返回。
7. 保证结果中元素顺序与输入中出现顺序一致。

完成标准：

1. 数组输入不会被修改。
2. 返回结果满足比较器定义的有序条件。
3. 返回结果是输入的子序列，而不是重排后的序列。

### 步骤 3：实现列表重载的原地压缩逻辑

目标：完成列表版本 `RemoveOutOfOrder` 的原地处理，尽量复用与数组版本一致的判定规则，同时保持列表 API 的使用体验。

实现要求：

1. 对 `values` 执行空值校验。
2. 使用与数组版本相同的顺序判定语义。
3. 通过写指针覆盖方式保留合法元素，避免逐项 `RemoveAt` 带来的重复移动成本。
4. 扫描完成后，调用一次 `RemoveRange` 删除尾部多余元素。
5. 如果列表原本已满足顺序要求，应尽量避免多余写入。
6. 保持保留元素的原始相对顺序。

完成标准：

1. 原地处理后的列表满足比较器定义的有序条件。
2. 列表内容是原输入的子序列。
3. 不引入额外公开辅助类型。

### 步骤 4：补齐测试用例并锁定公共语义

目标：在 `StrangeSortTest/StalinSort/` 下新增测试文件，覆盖输入校验、核心语义、自定义比较器和异常行为，确保后续重构不会破坏这套约定。

建议测试清单：

1. `null` 数组输入抛出 `ArgumentNullException`。
2. `null` 列表输入抛出 `ArgumentNullException`。
3. 空数组和单元素数组不要求默认比较器可用。
4. 空列表和单元素列表不要求默认比较器可用。
5. 已有序数组返回等价内容，且输入数组不变。
6. 已有序列表处理后内容保持不变。
7. 无序数组返回有序子序列，且输入数组不变。
8. 无序列表被原地修剪为有序子序列。
9. `null comparer` 与 `Comparer<T>.Default` 行为一致。
10. 自定义比较器可以定义不同顺序。
11. 逆序比较器可以得到降序子序列。
12. 默认比较器仅在真正需要比较时才抛异常。
13. 数组重载在比较器抛异常时不修改输入数组。
14. 列表重载在比较器抛异常时传播异常。
15. 语义锁定测试：`[3, 1, 2]` 必须得到 `[3]`。
16. 语义锁定测试：`[1, 3, 2, 4]` 必须得到 `[1, 3, 4]`。

完成标准：

1. 测试命名与现有测试风格一致。
2. 测试不依赖随机性。
3. 测试覆盖数组与列表两个公开入口。

### 步骤 5：执行验证并检查对外一致性

目标：通过构建和测试确认新增功能可用，并检查命名、注释和行为描述是否与既有代码风格一致。

验证要求：

1. 执行解决方案构建。
2. 执行测试项目全部测试。
3. 若新增注释或文档说明，与现有 `ThanosSorter` 的表述风格保持接近。
4. 确认没有意外扩大公开 API 面。

完成标准：

1. 构建通过。
2. 全量测试通过。
3. `StalinSorter` 的数组与列表行为符合本计划约定。

## 验收口径

功能完成后，应满足以下验收条件：

1. 仓库中存在新的 `StrangeSort.StalinSort` 命名空间与 `StalinSorter` 类型。
2. 用户可以对 `T[]` 和 `List<T>` 调用 `RemoveOutOfOrder`。
3. 数组调用后输入保持不变，返回值为有序子序列。
4. 列表调用后原列表被修剪为有序子序列。
5. 结果不会重排元素，只会删除违序元素。
6. 行为由测试稳定覆盖，尤其是 `[3, 1, 2] -> [3]` 与 `[1, 3, 2, 4] -> [1, 3, 4]` 这两个关键样例。
